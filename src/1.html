Dynamic Frontend System

Build websites and applications dynamically using a JSON-driven configuration

What is this project?

This project is a dynamic frontend framework that allows developers to create web applications where pages, layouts, components, and menus are defined entirely via a JSON configuration. It supports multiple layouts, role-based content, and pages without any layout.

Key Features

  • Dynamic Layouts: Admin, User, Public, or custom layouts.
  • No Layout Support: Fullscreen pages like login, editor, or print views.
  • JSON-Driven Components: Forms, tables, charts, cards, hero sections, and more.
  • Dynamic Menus: Menus change automatically depending on the user role.
  • Themes: Support for multiple themes (dark, light) configurable in JSON.
  • Extensible: New pages and components can be added by updating JSON only.

How It Works

  1. The frontend reads a master JSON configuration from backend or local file.
  2. Each page specifies a layout (admin, user, public, none).
  3. Components for each page are rendered dynamically based on their type and properties.
  4. If layout is "none", the page renders fullscreen without sidebars or headers.
  5. Menus and navigation are dynamically generated based on roles and JSON configuration.
  6. New pages or modifications require only JSON changes, no frontend code edits.

Sample JSON Configuration

{
  "layouts": {
    "public": { "theme": "light", "components": ["navbar", "footer"] },
    "user": { "theme": "light", "components": ["user-navbar", "user-footer"] },
    "admin": { "theme": "dark", "components": ["sidebar", "admin-topbar"] }
  },
  "pages": [
    {
      "path": "/home",
      "layout": "public",
      "title": "Home Page",
      "components": [
        { "type": "hero", "heading": "Welcome!", "subtext": "This is the public homepage" }
      ]
    },
    {
      "path": "/dashboard",
      "layout": "user",
      "title": "User Dashboard",
      "components": [
        { "type": "stats", "items": [
            { "label": "Orders", "value": 10 },
            { "label": "Messages", "value": 5 }
          ]
        }
      ]
    },
    {
      "path": "/admin",
      "layout": "admin",
      "title": "Admin Panel",
      "components": [
        { "type": "card", "heading": "Admin Stats", "content": "Some important admin info" }
      ]
    },
    {
      "path": "/login",
      "layout": "none",
      "fullscreen": true,
      "title": "Login Page",
      "components": [
        { "type": "form", "fields": [
            { "type": "text", "name": "email", "label": "Email" },
            { "type": "password", "name": "password", "label": "Password" }
          ],
          "buttons": [
            { "type": "submit", "label": "Login" }
          ]
        }
      ]
    }
  ]
}

Benefits

  • Speeds up frontend development and maintenance.
  • Separation of concerns: design vs. content configuration.
  • Supports multi-role applications and admin dashboards effortlessly.
  • Reduces repeated code: everything is controlled via JSON.

Conclusion

This dynamic frontend system provides a flexible, scalable, and maintainable approach for building modern web applications. Developers and designers can define complex applications without hardcoding layouts or pages, making updates and feature additions fast and safe.
